# Weave

Браузерный инструмент **Weave** для проектирования оптических сетей FTTH: OLT → муфта → ОРК → камера.
Работает полностью локально — без backend, без облака.

## Возможности

- 📂 Импорт абонентов из **Excel** (.xlsx) и **KMZ/KML**
- ➕ **Кумулятивный режим** — импортируй несколько городов в один проект
- 🤖 **Автопостроение** топологии (OLT → транзитные муфты → ОРК шкафы → дроп-кабели)
- 🛣 **OSRM роутинг** кабелей по реальным дорогам
- 🛰 **4 базовые карты**: тёмная / светлая / спутник / гибрид (со спутника видны дома)
- 📝 **Заметки на карте**: сёла, проблемы, зоны, точки фото, маршруты — точки/полигоны/линии/круги
- 📏 **Линейка** для измерения расстояний
- 🛠 **Ручное редактирование**: клик на карту = новый абонент
- 📊 **Статистика** — загрузка ОРК, разбивка по районам, валидация сети
- 📦 **Расчёт материалов** — кабели, сплиттеры, муфты, ONT, пигтейли, КДЗС, зажимы
- 💾 **Автосохранение** в localStorage + экспорт/импорт JSON проектов
- 📤 **Экспорт KMZ** с папками (импорт в Google Earth, GIS)
- 📊 **Экспорт Excel** (4 листа: материалы, абоненты, ОРК, кабели)
- ⌨️ **Горячие клавиши**

## Запуск локально

```bash
# Установка
npm install --legacy-peer-deps

# Режим разработки (рекомендуется для работы)
npm run dev
# Открой http://localhost:3000

# Production билд + запуск
npm run build
npm run start
```

## Деплой (без Vercel)

### Вариант 1: Просто запустить на своём ноутбуке
```bash
npm run build
npm run start
```
Приложение работает на `http://localhost:3000`. Все данные — в localStorage браузера.
**Это самый простой вариант** — не нужен сервер, не нужен интернет (кроме базовых карт).

### Вариант 2: Сделать доступным в локальной сети офиса
```bash
npm run build
npm run start -- -H 0.0.0.0 -p 3000
```
Коллеги в той же Wi-Fi сети могут зайти по адресу `http://<IP-твоего-компа>:3000`.

### Вариант 3: Docker
```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install --legacy-peer-deps
COPY . .
RUN npm run build
EXPOSE 3000
CMD ["npm", "run", "start"]
```
```bash
docker build -t weave .
docker run -p 3000:3000 weave
```

### Вариант 4: На своём VPS
```bash
git clone <repo>
cd telecom
npm install --legacy-peer-deps
npm run build
# Через PM2 для авто-перезапуска:
npm install -g pm2
pm2 start "npm run start" --name weave
pm2 save
pm2 startup
```
Поставить nginx для https и проксирования — стандартная схема.

### Вариант 5: Netlify / Cloudflare Pages
Подойдёт обычный фронтенд-деплой (Next.js поддерживается из коробки).

## Формат Excel для импорта

| Лист = район |  |  |
|---|---|---|
| **Latitude** | **Longitude** | **Description** |
| 40.777053 | 68.320873 | Жетісай қ., С.Қожанов/Әл-Фараби |
| 40.768823 | 68.318157 | Жетісай қ., М.Әуезов |

Каждый лист книги — отдельный район. Координаты в десятичных градусах.
Допустимый диапазон: lat 35–60, lon 45–90 (Казахстан и СНГ).

## Стек

- **Next.js 14** (App Router) + **TypeScript** + **Tailwind v4**
- **Leaflet.js** — карта (тёмная Carto, спутник Esri)
- **OSRM** — `router.project-osrm.org` (публичный, без ключа)
- **SheetJS** (xlsx) — импорт/экспорт Excel
- **JSZip** — импорт/экспорт KMZ
- **localStorage** — проекты

## Горячие клавиши

| Клавиша | Действие |
|---|---|
| `Ctrl+S` | Сохранить проект |
| `Ctrl+I` | Открыть импорт |
| `Ctrl+N` | Новый проект |
| `E` | Режим редактирования |
| `M` | Линейка |
| `?` | Справка |
| `Esc` | Отмена / закрыть |

## Замечания по работе с большими сетями

- При > 500 абонентов **отключи OSRM для дропов** (по умолчанию выкл.)
- Маркеры абонентов скрываются на zoom < 13 (для производительности)
- Дроп-кабели рисуются только на zoom ≥ 14
- localStorage хранит до 50 проектов — старые автоматически вытесняются
