# Self-host OSRM для прокладки кабеля

Публичный демо `router.project-osrm.org` отдаёт только профиль **driving** (машина).
Машина соблюдает односторонки и штрафует разворот → кабель «проезжает дальше и
разворачивается» и идёт по обеим сторонам дороги. Свой OSRM с пешеходным профилем
(**foot**) это убирает: пешеход ходит в обе стороны по одним рёбрам, без объездов,
поэтому туда и обратно ложатся одной линией.

> Это НЕ переписывание OSRM — мы запускаем официальный образ и лишь берём другой
> профиль (`/opt/foot.lua` уже внутри образа).

## Требования
- Docker + docker compose
- ~2–4 ГБ диска под Казахстан (меньше, если обрезать до города)

## Шаги

```bash
cd osrm
./setup.sh          # скачать регион + extract/partition/customize (профиль foot)
docker compose up -d # поднять сервер на http://localhost:5000
```

Проверка:
```bash
curl 'http://localhost:5000/route/v1/foot/76.9,43.25;76.95,43.25?overview=false'
# ожидаем "code":"Ok"
```

## Подключение к приложению

Есть два способа указать адрес OSRM. Первый — без пересборки, его и рекомендуем
для бесплатного туннеля.

### Способ 1 (рекомендуется): поле в приложении

Открой вкладку **Tools → Маршрутизация кабелей → OSRM-сервер**, впиши адрес
своего OSRM и выбери профиль `foot`, нажми «Сохранить». Применяется при следующей
прокладке, **пересборка не нужна** — удобно, когда URL туннеля меняется.

### Способ 2: env при сборке

В `.env.local` (`NEXT_PUBLIC_*` инлайнятся при сборке):

```
NEXT_PUBLIC_OSRM_HOST=http://localhost:5000
NEXT_PUBLIC_OSRM_PROFILE=foot
```

```bash
npm run build && npm run start   # или перезапустить dev-сервер
```

Приоритет: поле в приложении → env → дефолт (публичный demo/driving).

## Бесплатно: свой компьютер + Cloudflare Tunnel

Прод на Vercel ходит по https, поэтому `http://localhost:5000` из браузера не
подойдёт (mixed content). Бесплатно отдать локальный OSRM наружу по https можно
через Cloudflare Tunnel — карта и аккаунт не нужны:

```bash
# 1. локальный OSRM (foot) уже поднят: docker compose up -d  → localhost:5000
# 2. установить cloudflared и поднять быстрый туннель:
cloudflared tunnel --url http://localhost:5000
```

`cloudflared` напечатает случайный адрес вида `https://<random>.trycloudflare.com`.
Скопируй его в поле **OSRM-сервер** (Способ 1), профиль `foot`, «Сохранить».

> Минусы быстрого туннеля: работает только пока твой ПК и `cloudflared` включены,
> а URL меняется при каждом перезапуске — поэтому и используем поле в приложении,
> а не env (вписал новый адрес за секунду, без пересборки). Для постоянного адреса
> нужен именованный туннель на своём домене в Cloudflare.

## Тонкая настройка

- **Быстрее сборка** — обрежьте регион до города (`osmium extract`) и положите как
  `data/region.osm.pbf`, затем `./setup.sh` (скачивание пропустится).
- **Свой профиль кабеля** — вместо `foot` можно подсунуть свой `*.lua` (все дороги
  двусторонние, вес = длина) через `PROFILE=...`, положив файл рядом и смонтировав в
  образ. Для 95% случаев штатного `foot` достаточно.
- **Прод** — поднимите этот же сервис на сервере и укажите его URL в
  `NEXT_PUBLIC_OSRM_HOST` (через https/reverse-proxy).
