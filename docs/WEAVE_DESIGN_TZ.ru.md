# ТЗ на дизайн сайта Weave (для Claude Designer / Figma AI)

**Заказчик:** инженерная команда GPON/FTTH (Казахстан, проекты Sergek / видеонаблюдение).  
**Продукт:** **Weave** — веб-редактор оптической сети на карте (Next.js + Leaflet).  
**Цель ТЗ:** получить **красивый, современный, профессиональный UI**, который не ломает сложную карту и десятки инженерных функций.

---

## 1. Контекст и позиционирование

| Параметр | Значение |
|----------|----------|
| Название | **Weave** |
| Слоган (EN) | *From OLT to camera — one road, one line.* |
| Слоган (RU, предложить вариант) | *От OLT до камеры — одна дорога, одна линия.* |
| Суть | Проектирование волокна: OLT → муфта → ОРК → боксы → камеры; маршрут по дорогам (OSRM); смета и KMZ |
| Аудитория | Инженеры связи, проектировщики, сметчики; русский / казахский интерфейс |
| Платформа | Desktop-first (1920×1080), минимум 1280px; планшет — вторично; телефон не приоритет |
| Хостинг | Vercel, тёмная тема по умолчанию (карта ночью) |

**Образ бренда:** точность инженерии + «чистая нить» на карте. Не игровой стиль, не корпоративный «банк 2015». Ближе к: **Linear**, **Figma dark**, **Mapbox Studio**, **Railway** — но со своим акцентом (волокно / свет / дорога).

---

## 2. Проблемы текущего UI (что исправить)

Сейчас интерфейс **функционален**, но выглядит как прототип:

- В шапке **15+ кнопок** в один ряд с эмодзи — перегруз, нет группировки.
- Боковая панель: **10 вкладок** с подписями 8px — трудно читать.
- Везде **emoji как иконки** (📡 📂 💾) — несерьёзно для B2B-сметы.
- Одинаковые `border-[#1e3a5f]` рамки — всё сливается.
- Нет **логотипа** Weave (только текст + 📡).
- Нет **onboarding** и визуальной иерархии «сначала импорт → ① → ②».
- Модалки импорта/справки — плотный текст без визуальных шагов.
- Прогресс OSRM — ок, но можно сделать **stepper** «Импорт → Построение → Проход 1 → Проход 2 → Экспорт».

**Не трогать в визуальном ТЗ (логика остаётся):** слои кабелей на карте (цвета ОК-4/8/12/16), маркеры OLT/муфта/ОРК, Leaflet.

## 3. Структура приложения (карта экранов)

```
┌─────────────────────────────────────────────────────────────────┐
│ TOP BAR: лого | проект | статус | KPI | поиск | действия        │
├──────────┬──────────────────────────────────────────────────────┤
│ SIDEBAR  │ MAP (Leaflet, 70–80% ширины)                          │
│ 280–320px│  + floating: прогресс, empty state, контекст-меню    │
│ tabs     │  + overlays: EntityEditor, CableEditor, Chat (drawer)│
└──────────┴──────────────────────────────────────────────────────┘
```

### 3.1. Главный экран (единственный route `/`)

**A. Top bar (высота 48–56px)**  
Секции слева направо:

1. **Brand** — логотип Weave (знак: переплетённые линии / монофиламент / буква W как траектория кабеля).
2. **Проект** — название (editable), статус (черновик / в работе / согласован / сдан), индикатор сохранения.
3. **KPI chips** — абоненты, км кабеля, ОРК, предупреждения (клик → вкладка Статистика).
4. **Поиск** — геокод (уже есть GeocodeSearch).
5. **Primary actions** — сгруппировать:
   - **Файл:** Импорт, Сохранить, Проекты
   - **Сеть:** Построить / Перестроить кабели (контекстно)
   - **Редактирование:** режим Просмотр/Редакт, undo/redo
   - **Размещение:** OLT, Муфта, ОРК, Кабель (toolbar второго уровня или flyout)
   - **More:** Каталог, Камеры, Выделение, Справка, AI-чат

**B. Sidebar (левая колонка)**  
Переработать навигацию — **не 10 равных иконок**, а:

- **Вариант 1 (рекомендуется):** 4 группы с accordion:
  - *Карта и слои* (слои, заметки)
  - *Сеть и смета* (материалы, ОРК-группы, схема)
  - *Анализ* (статистика, бюджет, цена)
  - *Проект* (инструменты ①②, экспорт, проекты, снимки)
- **Вариант 2:** вертикальный rail (иконки) + контент панели справа от rail (как VS Code).

Вкладки по смыслу (сохранить все функции):

| ID | Название RU | Содержимое |
|----|-------------|------------|
| layers | Слои | OLT, муфты, ОРК, кабели по типам ОК |
| notes | Заметки | Аннотации на карте, рисование |
| materials | Материалы | Метраж, муфты, сплиттеры |
| groups | ОРКСП | Группы камер по шкафам |
| schema | Схема | Дерево OLT → … |
| stats | Статистика | Валидация, ошибки |
| budget | Оптика | Power budget |
| cost | Стоимость | Прайс, итог |
| tools | Инструменты | ① OSRM, ② слияние, heatmap, PDF |
| projects | Проекты | Cloud/local, snapshots |

**C. Карта**  
- Тёмная подложка (сохранить).  
- **Легенда** кабелей — компактная, сворачиваемая, правый нижний угол.  
- **Масштаб / координаты** — стиль Mapbox.  
- **Empty state** — hero с 3 шагами: Импорт → Построить → Проход 1 и 2; CTA крупные, без emoji-гигантов.

**D. Floating / overlay**  
- Прогресс маршрутизации — stepper + progress bar (уже есть логика).  
- **Entity editor** — карточка справа (OLT/Муфта/ОРК), не «форма в углу».  
- **Chat AI** — drawer справа 360px, не перекрывать KPI.  
- **Контекстное меню** карты — иконки + подписи.

### 3.2. Модальные окна (редизайн оболочки, контент тот же)

| Модалка | Назначение |
|---------|------------|
| ImportModal | Excel/KML/KMZ, настройки стороны дороги |
| AddCamerasModal | Добавление камер к сети |
| ProjectListModal | Список проектов cloud |
| CatalogModal | Каталог оборудования |
| Help | Справка + hotkeys |
| SplicePlan | План сварки муфты |

**Требования к модалкам:** ширина max 560–720px, заголовок + шаги (step indicator для импорта), footer с primary/secondary, фокус-trap, ESC закрывает.

### 3.3. Маркетинг (опционально, если Designer делает лендинг)

Отдельная страница **не в scope v1** — только если есть время: hero, 3 feature cards, скриншот карты, CTA «Открыть редактор».

## 4. Визуальное направление (на выбор дизайнера — 2 концепции)

Предложить **2 moodboard-концепции** в Figma, заказчик выберет одну.

### Концепция A — «Fiber cyan» (эволюция текущего)

- Фон: `#080c14` / `#0c1220`
- Поверхности: `#111827` с лёгким blur
- Акцент: cyan `#22d3ee` → primary; amber `#fbbf24` — предупреждения; emerald `#34d399` — успех
- Типографика: **Inter** (UI) + **JetBrains Mono** (координаты, метраж)
- Радиусы: 8px кнопки, 12px карточки, 16px модалки
- Тени: мягкое свечение акцента `0 0 24px rgba(34,211,238,0.15)`

### Концепция B — «Copper & graphite»

- Фон графит, акцент тёплый медный/янтарный (волокно подсвечено)
- Карта: чуть теплее подложка, кабели — холодные линии контрастом
- Более «инженерный» вид для заказчиков строительства

**Общее для обеих:** контраст WCAG AA для текста, disabled-состояния явные, hover 150ms ease.

## 5. Дизайн-система (deliverables)

Дизайнер **обязан** отдать:

1. **Figma file** (или эквивалент) с:
   - Color tokens (CSS variables)
   - Typography scale: 10 / 11 / 12 / 14 / 16 / 20 / 24
   - Spacing: 4px grid
   - Components: Button (primary, secondary, ghost, danger), Input, Select, Chip, Tab, Accordion, Card, Modal, Toast, Tooltip, Progress, Stepper, KPI badge, Sidebar item, Map legend
2. **Иконки:** SVG set 20×20 и 24×24 — **без emoji** (Lucide/Heroicons style, единый stroke 1.5px)
3. **Логотип Weave:** full + mark, светлый на тёмном фоне
4. **3 ключевых экрана hi-fi:**
   - Empty state + sidebar
   - Сеть построена + прогресс OSRM
   - Импорт (модалка шаг 2)
5. **UI kit для разработчика:** таблица «компонент → Tailwind class / CSS var»

### Цвета кабелей на карте (не менять без согласования)

| Тип | Текущий смысл |
|-----|----------------|
| ОК-4 | тонкая ветка, дальние абоненты |
| ОК-8 … ОК-16 | рост к OLT |
| OLT / муфта / ОРК | маркеры узлов |

## 6. UX-сценарии (прототип flows)

Прорисовать **user flow** (можно в FigJam):

1. **Новый проект:** empty → Импорт Excel → Построить → badge «Запустите ①» → Tools → ① → ② → Экспорт KMZ.
2. **Правка муфты:** клик на муфту → EntityEditor → Сохранить → «Перестроить кабели» в top bar (highlight) → снова ①.
3. **Ошибки:** KPI «⚠ 12» → Stats → список с fly-to на карте.
4. **AI:** кнопка в header → drawer, не блокирует карту.

**Микрокопирайтинг RU:** короткие глаголы («Импортировать», «Прокласть по дорогам», «Объединить линии»), не «Проход 1» без пояснения в tooltip.

## 7. Технические ограничения для вёрстки

- Стек: **Next.js 15**, **Tailwind CSS v4**, `globals.css` с CSS variables.
- Нельзя: тяжёлые UI-библиотеки без согласования (MUI/Chakra — только если kit лёгкий).
- Карта: Leaflet — **не redesign tiles**, только chrome вокруг.
- Шрифты: Google Fonts CDN (как сейчас) или self-host.
- Печать карты: `@media print` — предусмотреть светлый вариант линий (опционально v2).

---

## 8. Критерии приёмки дизайна

- [ ] Top bar ≤ 2 визуальных ряда на 1440px, без горизонтального скролла
- [ ] Все 10 функций sidebar доступны ≤ 2 клика
- [ ] Empty state объясняет pipeline за 5 секунд
- [ ] Есть логотип + favicon 32/180
- [ ] Иконки единый стиль, 0 emoji в production UI
- [ ] Primary CTA «Импорт» визуально доминирует
- [ ] Кнопки ① и ② в Tools — визуально связаны (stepper / numbered pills)
- [ ] Модалка импорта — wizard ≤ 4 шага
- [ ] Dark theme контраст ≥ 4.5:1 для body text
- [ ] Handoff: Figma Dev Mode или PDF spec + экспорт SVG logo/icons
