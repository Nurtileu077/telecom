# Sergek GPON — чеклист топологии

Обновлено при исправлении цепочки BOX + консолидации.

## P0 — топология Sergek (схема заказчика)

- [x] Каскад **1:8 × 1:8 = 64** (по умолчанию L1 1:8)
- [x] **OLT → муфта:** ОК-16 (16F, не 12F/48)
- [x] **Муфта → ОРКСП:** ОК-4 (2F: 1 раб. + 1 рез.)
- [x] **ОРКСП → 8 камер:** ОК-16 по цепочке боксов
- [x] ОРКСП на первой камере цепочки, greedy NN
- [x] Consolidation: путь через BOX + **потолок ОК-16** (не N×2→ОК-48)
- [x] `mergeCorridorM` 12 м, OSRM snap OntBox

## Известные ограничения

- Магистраль OLT→Муфта на полный порт (до 64 камер) по правилу N×2 может быть **ОК-48** — это потолок проекта (ОК-96 запрещён).
- Inline-муфты не дублируют L1-муфту и ОРКСП на карте — отдельные сущности.
