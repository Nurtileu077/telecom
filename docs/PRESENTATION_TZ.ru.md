# OPTIQ — техническое задание на презентацию для руководства

**Продукт:** OPTIQ (Weave / GPON Network Designer)  
**Слоган:** «От OLT до камеры — одна дорода, одна линия»  
**Аудитория презентации:** руководство, технические директора, заказчик проектирования оптики (Sergek / видеонаблюдение на оптике)  
**Формат итогового материала:** красивая презентация (слайды), язык — русский, тон — деловой но понятный  

---

## 1. О чём продукт (одним абзацем)

**OPTIQ** — веб-приложение для проектирования оптических сетей FTTH/GPON: от импорта координат камер (абонентов) до прокладки кабелей по дорогам, расчёта материалов, оптического бюджета и выгрузки в KMZ/PDF/Excel. Работает в браузере и как **PWA** («На экран Домой» на телефоне) — полевой инженер и проектировщик видят одну карту, одну сеть, один проект.

---

## 2. Для кого и какую боль закрывает

| Роль | Задача | Как помогает OPTIQ |
|------|--------|-------------------|
| Проектировщик | Быстро набросать сеть по Excel/KMZ | Импорт → автопостроение → OSRM по дорогам |
| Инженер в поле | Проверить муфту/ОРК на месте | Телефон: карта + «паспорт узла» + координаты |
| Руководитель проекта | Смета, статус, отчёт | Материалы, стоимость, PDF, статусы проекта |
| Согласующий | Понять сложность узла | «Геймификация» сложности, бюджет затухания |

## 3. Что умеет сайт (веб-версия, полный функционал)

### 3.1. Карта и навигация

- Интерактивная карта (Leaflet): тёмная тема, масштабирование, поиск адреса (геокодинг).
- Отображение: **OLT**, **муфты (TB)**, **ОРК**, **камеры/абоненты**, кабели с типами ОК-4…ОК-48, транзитные муфты на развилках.
- Слои вкл/выкл: OLT, муфты, ОРК, абоненты, типы кабелей, заметки.
- **Линейка** — измерение расстояния на карте.
- **Тепловая карта** загрузки ОРК (сколько камер на бокс).
- Подсветка **ветки** от выбранного OLT/муфты/ОРК/камеры.
- Режимы **просмотр / редактирование**.

### 3.2. Импорт и старт проекта

- **Excel** — координаты и типы камер (ЛУ, перекрёсток, ОВН), районы.
- **KMZ/KML** — структурированный импорт точек и линий.
- **Кумулятивный импорт** — несколько файлов/городов в один проект.
- **Добавить камеры в существующую сеть** (brownfield Excel) — привязка к ближайшему ОРК + OSRM.
- Пустой проект → мастер из 3 шагов: Импорт → Построение → Маршрут.

### 3.3. Автопостроение сети

- Кластеризация абонентов по районам.
- Размещение **OLT → муфты → ОРК → дроп** к камерам.
- Подбор типов кабелей по числу абонентов (без ОК-96 по политике проекта).
- **Слияние параллельных кабелей** на одной трассе (визуально отдельные «полосы»).
- Транзитные муфты на развилках магистрали (авто-узлы на кабеле).

### 3.4. Маршрутизация OSRM

- Прокладка кабелей **по дорогам** (router.project-osrm.org / Night Fiber).
- Прогресс: процент, счётчик кабелей, текущая операция, кнопка **Остановить**.
- Перепрокладка одного кабеля или всей сети / по **лассо-выделению**.
- Привязка новых узлов к ближайшей дороге (snap).

### 3.5. Ручное проектирование

- Постановка **OLT, муфты, ОРК** кликом на карту.
- Рисование кабеля **между узлами** (Кабель) или **A→B** по точкам на карте.
- Режим **«Соединить»** — клик по кабелю, затем по узлу; подсветка совместимых целей.
- **Магнитная привязка** при перетаскивании узлов и концов кабелей.
- Редактирование трассы кабеля: **опорные точки** (waypoints), перетаскивание без «каляки».
- Перемещение OLT/муфты/ОРК на карте (режим move).
- Контекстное меню на карте: добавить абонента/OLT/муфту/ОРК.
- **Лассо** — полигональное выделение для экспорта/перепрокладки/консолидации в зоне.

### 3.6. «Что внутри» узла (паспорт объекта) — ключевая фишка UX

При клике на **OLT / муфту / ОРК** справа открывается панель (карта остаётся видимой):

- **Координаты** (копирование, показ на карте).
- **Центр:** схема внутри узла.
- **Низ:** свойства, соединения-ссылки, действия.

**Муфта:** все кабели, таблица «волокно → куда» (сварки), переход в ОРК.  
**ОРК:** посты сплиттера; тап по посту → **камера**, тип (ЛУ/перекрёсток/ОВН), **затухание dB**, переход на карту.  
**OLT:** дерево муфт → ОРК, L1-сплиттер.  
**Транзитная муфта** (⊕ на развилке): тоже открывается, не только «МТОК-боксы».  
**Сложность узла:** шкала 0–100, звёзды, уровень (Новичок → Магистр), подсказки.

### 3.7. Оптика и анализ

- **Оптический бюджет** по каждой камере (затухание, RX, ok/warn/fail).
- Раскраска абонентов по бюджету на карте.
- Вкладки: **Статистика**, **Бюджет**, **Стоимость** (прайс, монтаж).
- **Материалы:** кабели, сплиттеры, муфты, ONT, расходники.
- **Схема** сети, группы **ОРК**.
- **Валидация** перед экспортом (ошибки/предупреждения).

### 3.8. Заметки и совместная работа с картой

- Аннотации: точки, линии, полигоны, круги (сёла, проблемы, фото, зоны).
- Инструменты рисования на вкладке «Заметки».

### 3.9. Проекты и данные

- Сохранение проекта (localStorage + опционально Supabase).
- **Автосохранение**, история импортов.
- **Снимки** (snapshots) — откат к версии.
- **Undo/Redo** с подписью действия («Отменить: Кабель OLT→TB»).
- Статусы проекта: Дизайн → На согласовании → Утверждён → Строительство → В эксплуатации.
- Экспорт/импорт **JSON** проекта.

### 3.10. Экспорт

- **KMZ** (Google Earth, GIS), с папками и подписями.
- **Excel** (материалы, абоненты, ОРК, кабели).
- **PDF** отчёт (с проверкой перед выгрузкой; можно по лассо-зоне).
- Печать карты.

### 3.11. AI-ассистент (опционально)

- Чат с контекстом сети: вопросы, подсказки, часть действий на карте (если настроен API).

### 3.12. Каталог и типы камер

- Типы: **ЛУ** (~26 Мбит/с), **Перекрёсток** (~78), **ОВН** (~5) — влияют на ёмкость и цвет на карте.
- Каталог оборудования/цен (при включённой БД).

## 4. PWA — приложение «На экран Домой» (мобильная версия)

### 4.1. Что это для пользователя

- Иконка **OPTIQ** (логотип без текста на картинке) на рабочем столе iPhone/Android.
- Открывается **без адресной строки**, как нативное приложение (`display: standalone`).
- Под иконкой может быть подпись «OPTIQ» — это имя ярлыка, не рисунок на иконке.

### 4.2. Мобильный интерфейс

**Верх:** компактная шапка — меню ☰, логотип, название проекта, статус, сохранение, **Импорт**, AI.  
**Низ:** панель как в приложении:

| Кнопка | Действие |
|--------|----------|
| **Домой** | Карта, закрыть меню и панели |
| **Меню** | Боковой drawer: слои, материалы, инструменты… (меню **не закрывается** при переключении вкладок) |
| **+** | Быстрые действия: импорт, OLT, муфта, ОРК, кабель, A→B |
| **Поделиться** | Системное «Поделиться» или копирование ссылки |

**Паспорт объекта на телефоне:** снизу листом; в центре — «что внутри»; сверху координаты.  
**Прогресс OSRM:** фиксированная плашка **над** нижним доком (процент, полоса, остановить).  
**Safe area:** учёт выреза iPhone (статус-бар, home indicator).  
Баннер «Установить как приложение» (подсказка для iOS: Поделиться → На экран Домой).

### 4.3. Что на телефоне урезано / неудобно (честно для слайда «ограничения»)

- Мелкие кнопки шапки на узком экране — часть инструментов только в меню или на планшете.
- Точное редактирование waypoints кабеля на пальце сложнее, чем мышью на ПК.
- OSRM на тысячах кабелей — долго; на телефоне важно не блокировать экран (сейчас есть оверлей прогресса).
- Нет офлайн-карты без интернета (нужен интернет для тайлов и OSRM).

## 5. Удобства для руководства и команды (продать на слайде)

1. **Одна линия правды** — от импорта до KMZ/PDF без выгрузки в 5 программ.  
2. **Дороги, не прямые линии** — кабели по OSRM, меньше сюрпризов на стройке.  
3. **Понятно «что внутри муфты»** — не чёрный ящик для монтажника.  
4. **Затухание на камеру** — риск перегруза виден до монтажа.  
5. **Поле + офис** — тот же проект в браузере и на телефоне (PWA).  
6. **Статус проекта** — где дизайн, где стройка, где эксплуатация.  
7. **Откат и снимки** — меньше страха «сломать проект».  
8. **Лассо** — работать по кварталу/району, не по всему городу.  
9. **Смета из проекта** — материалы и стоимость из той же модели.  
10. **Бренд OPTIQ** — единый тёмный UI, узнаваемый логотип.

## 6. Было / стало — прошлые минусы и что исправлено

Использовать в презентации как блок «Эволюция продукта».

| № | Было (боль) | Стало |
|---|-------------|--------|
| 1 | Кабель тянулся «змейкой» при редактировании | Жёсткие опорные точки, snap, без лишних вставок |
| 2 | Клик по узлу при рисовании кабеля открывал инспектор вместо выбора конца | Режимы «Кабель» / «Соединить» / A→B разделены |
| 3 | На телефоне всё наезжало, меню закрывалось на каждый тап | Drawer + док + safe area + полноэкран на ПК |
| 4 | «Внутри муфты» только у части узлов, только полноэкран | OLT/ОРК/муфта/транзит; панель справа, карта видна |
| 5 | Иконка PWA со словом OPTIQ, старая графика | Логотип как на сайте, static apple-touch-icon |
| 6 | Верх/низ уехали под статус-бар на iPhone | Safe-area, исправлена высота 50% на десктопе |
| 7 | Прогресс OSRM на телефоне не виден | Fixed-оверлей над доком, проценты всегда |
| 8 | Не все муфты на карте открывались | Клик по транзитным ⊕ на развилках |
| 9 | Нет координат при выборе объекта | Блок «Координаты» в паспорте, копирование |
| 10 | Пост ОРК не вёл к камере на карте | Тап по посту → flyTo камера + затухание |

**Остающиеся ограничения (не скрывать):**

- Зависимость от внешнего OSRM (лимиты, сеть).
- Очень большие города — время построения и роутинга.
- Нет полноценного offline-first PWA (service worker не заявлен).
- AI требует ключ API.
- Supabase/каталог — опционально, основной режим — браузер + localStorage.

## 8. Архитектура в двух словах (для IT-руководства)

- **Frontend:** Next.js 14, React, Leaflet, TypeScript.  
- **Данные:** модель District/OLT/TB/ORK/Subscriber/Cable в памяти; persist localStorage (+ optional Supabase).  
- **Роутинг:** OSRM API, батчами с прогрессом.  
- **Деплой:** статический/Node `next start`, Docker, VPS; URL примера: `https://optiq.khan-engineering.net/`.  
- **Без обязательного backend** для ядра — можно в закрытой сети офиса.

## 10. Полная справка по приложению (пользовательская)

### 10.1. Первый запуск

1. Открыть сайт или установить PWA.  
2. **Импорт** — Excel или KMZ с координатами.  
3. **Построить** — автоматическая топология (если кнопка доступна).  
4. Дождаться **«Прокладка по дорогам»** (OSRM).  
5. Проверить слои, бюджет, экспорт.

### 10.2. Горячие клавиши (ПК)

| Клавиша | Действие |
|---------|----------|
| Ctrl+Z | Отмена |
| Ctrl+Shift+Z | Повтор |
| E | Режим редактирования |
| M | Линейка |
| Escape | Закрыть модалки, снять выделение |
| ? | Справка |

### 10.3. Левая панель — разделы

- **Карта → Слои:** видимость OLT, муфт, кабелей, абонентов.  
- **Карта → Заметки:** рисование на карте.  
- **Сеть → Материалы / ОРК / Схема.**  
- **Анализ → Статистика / Бюджет / Стоимость.**  
- **Процесс → Инструменты** (OSRM, лассо, PDF, тепловая карта) **/ Проекты.**

### 10.4. Верхняя панель (ПК)

Просмотр/Редакт., Undo/Redo, +OLT, Муфта, ОРК, Кабель, A→B, Соединить, Лассо, Камеры, Проекты, Сохранить, **Импорт**, AI.

### 10.5. Работа с объектом

- **Клик** — выбор + паспорт справа + «что внутри».  
- **Двойной клик** — режим перемещения на карте.  
- **ПКМ** — контекстное меню.  
- В паспорте: координаты, свойства, **Показать ветку**, **Переместить**, удаление.

### 10.6. Кабели

- Тип ОК-4…ОК-48, длина, привязка from→to.  
- Редактор кабеля: тип, OSRM одного кабеля, waypoints, соединить конец, удалить.

### 10.7. Экспорт и проверки

- Перед PDF — валидация (ошибки/предупреждения).  
- Лассо ограничивает зону экспорта PDF.  
- KMZ/Excel — полный проект или по логике инструмента.

### 10.8. Мобильное (PWA)

- Установка: Safari → Поделиться → На экран Домой.  
- Работа: карта + док; меню для слоёв; **+** для добавления; паспорт объекта снизу/справа.  
- При маршрутизации — не закрывать вкладку; смотреть плашку прогресса.
